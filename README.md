# 墨灵 - AI小说创作工具

一款会随着创作者一同成长的AI小说创作助手，将AI的发散性与生成力，与作家的掌控感与判断力完美结合。

## 🌟 核心理念

### 三层创作模式

- **灵感漫游模式**：无压力的创意起点，突破"不知道写什么"
- **结构化创作模式**：从大纲、人物到章节，逐步推进，确保长篇不崩
- **自由沙盒模式**：完全手动控制，AI按需召唤，给资深作者最大的自由度和掌控感

## 🎯 核心功能模块

### 1. 灵感孵化室

解决创作"冷启动"难题：
- **脑洞聊天**：输入模糊想法，AI以提问方式帮忙拓展
- **"如果"生成器**：用经典故事模板随机组合元素，生成颠覆性开局
- **热点/经典融合器**：输入喜欢的两个作品，AI分析核心魅力点并生成融合新故事
- **核心创意提取器**：上传随笔、梦境记录或新闻，AI提取可发展成小说的核心冲突、独特设定和人物原型
- **类型宝典**：内置悬疑、言情、科幻等类型"故事引擎"，解析必备桥段、节奏、爽点和雷区

### 2. 人物工坊

让角色真正"活"起来：
- **多维档案生成**：输入标签，自动生成童年经历、口癖、内心矛盾、秘密等完整档案
- **可视化关系网**：以图谱展示人物关系，修改角色目标时自动预警可能的人物性格冲突
- **内生小说引擎**：将建好的人物放入虚拟场景，AI模拟基于性格和动机的真实对话与心理活动
- **人物弧光追踪**：标注每个人物在三幕结构中的成长变化，角色被遗忘或停滞时自动提醒

### 3. 世界观/设定集

为幻想类作者打造的"创世神器"：
- **逻辑一致性守卫**：记录设定规则，一旦后文出现矛盾立即划线提示
- **交互式地图与编年史**：文字设定一键生成可视化时间线或地图，点击地点即可看到相关故事片段和人物

### 4. AI辅助写作区

核心创作界面：
- **动态大纲与回溯织网**：大纲非静态，插入新伏笔时自动建议在前文埋下呼应细节，故事线联动如网
- **多维续写引擎**：卡文时指定参数生成续写建议（风格、参照、节奏），每次提供3-5个不同方向（保守/意料之中/极度出人意料）
- **情感与节奏调色板**：选中文字，用滑块调整"叙事节奏"或"情感浓度"，AI相应重写段落
- **伏笔与线索管理器**：高亮标记伏笔，设置"揭晓时间"，全程跟踪确保无坑，接近揭晓点时提醒

### 5. 智能编辑与风格打磨

- **多语言/风格翻译器**：把一段文字翻译成"海明威式英文"或"普鲁斯特式长句"
- **信息密度调节器**：用滑块把冗长描写浓缩为精辟比喻，或把短句展开为史诗段落
- **对话抛光机**：智能检测对话段落，优化台词使之更符合人物口癖、增加潜台词、强化张力
- **一致性守卫（终审）**：扫描全文，列出所有事实、性格、设定上的潜在冲突

### 6. Skills进化技能系统

这是墨灵区别于所有静态AI写作工具的根本特征：

- **对白雕刻师**：生成符合角色性格的对话 → 逐渐掌握角色独特口癖、潜台词偏好、言语交锋节奏
- **场景构建师**：根据提示渲染环境氛围 → 学会用户偏好的感官描写顺序、光影隐喻风格
- **伏笔织网者**：识别并记录伏笔 → 主动建议在现有章节埋下与新剧情呼应的彩蛋
- **节奏鼓手**：调整段落节奏 → 学会该小说类型在该阶段的"黄金节奏比"
- **文风变奏者**：模仿指定作家或风格 → 融合用户上传的个人手稿，生成独属于该用户的"成长中的私人文风模型"
- **角色灵魂绑定**：维持角色性格一致性 → 进化到能模拟角色内心独白的"第一人称日记"

## 🛠️ 技术栈

- **框架**：Tauri + React 18 + TypeScript
- **状态管理**：Zustand
- **UI样式**：TailwindCSS 3
- **图标**：Lucide React
- **本地存储**：IndexedDB via localForage
- **AI服务**：OpenAI API (gpt-4o-mini)

## 📦 安装使用

### 前置要求

- Node.js 18+
- pnpm 或 npm/yarn
- Rust (用于Tauri桌面应用构建)

### 开发模式

```bash
# 安装依赖
pnpm install

# 启动开发服务器
pnpm run dev

# 构建生产版本
pnpm run build
```

### 桌面应用构建

```bash
# 开发模式（桌面）
pnpm run tauri dev

# 构建桌面应用安装包
pnpm run tauri build
```

## 📝 配置说明

### AI配置

在应用的设置页面中配置：
1. 输入你的OpenAI API Key
2. 选择AI模型（默认使用 gpt-4o-mini）
3. 调整温度等生成参数

### 本地存储

所有数据默认存储在本地IndexedDB中，支持离线使用：
- 小说数据
- 人物设定
- 世界观设定
- Skills进化状态

## 🚀 用户工作流示例

### 初次创作完整流程

1. 启动 → 进入"灵感漫游模式"
2. 灵感碰撞 → 与AI脑洞聊天，完善核心创意
3. 奠基 → 切换到"结构化模式"，一键导入人物工坊和世界观设定
4. 大纲 → 使用"类型宝典"参考模板，与AI协作完成三幕式章节大纲
5. 创作 → 在写作区使用多维续写引擎逐章推进，必要时启动内生小说引擎观察角色互动
6. 精修 → 运行一致性守卫修复设定bug，用节奏调色板打磨关键段落
7. 导出 → 一键排版为投稿格式或精美预读本

### Skills进化三阶段体验

- **初期（1-10章）**：主动训练期 → 频繁拖拽基础Skills到续写指令区，从多个结果中挑选并修改
- **中期（10-30章）**：默契协作期 → AI续写建议默认风格已非常对味，简单指令即可获得基本无需大改的段落
- **后期（30章后）**：灵犀直觉期 → AI不待指令主动提供建议，如"检测到关键转折，建议此处引用第3章意象形成呼应"

## 📥 下载安装

项目使用GitHub Actions自动构建，每次推送到main分支都会自动构建各平台安装包。

### 下载地址

访问 [GitHub Releases](https://github.com/Qiu-Shiyi/Moling-Novel-writer/releases) 页面下载最新版本：

如果你想自己构建：

```bash

# 安装依赖
pnpm install

# 构建生产版本
pnpm run build
```
